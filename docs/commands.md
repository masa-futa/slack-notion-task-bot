## /task - タスク追加

新しいタスクを Notion に追加します。

### 基本構文

```
/task タスク名 [@優先度] [#カテゴリ] [due:期限]
```

### パラメータ

| パラメータ | 必須 | 形式 | 説明 |
|-----------|------|------|------|
| **タスク名** | ✅ | テキスト | タスクの内容 |
| **優先度** | ⚠️ | `@高` `@中` `@低` | デフォルト: 中 |
| **カテゴリ** | ⚠️ | `#学習` `#開発` `#生活` `#趣味` | 省略可 |
| **期限** | ⚠️ | `due:日付` | 省略可 |

### 優先度の指定

日本語または英語で指定可能:

```bash
@高 / @high     # 🔴 高
@中 / @medium   # 🟡 中
@低 / @low      # 🟢 低
```

### カテゴリの指定

日本語または英語で指定可能:

```bash
#学習 / #study  # 📚 学習
#開発 / #dev    # 💻 開発
#生活 / #life   # 🏠 生活
#趣味 / #hobby  # 🎨 趣味
```

### 期限の指定

複数の形式をサポート:

```bash
due:今日       # 今日
due:tomorrow  # 明日
due:金曜      # 次の金曜日
due:Friday    # 次の金曜日
due:2026-05-01 # 具体的な日付
```

---

### 使用例

#### 基本

```
/task ブログ記事を書く
```

→ 優先度「中」、カテゴリなし、期限なしで追加

#### 優先度付き

```
/task 重要な会議の準備 @高
/task 本を読む @低
```

#### カテゴリ付き

```
/task Flutter を勉強する #学習
/task API を実装する #開発
/task 掃除をする #生活
```

#### 期限付き

```
/task レポート提出 due:明日
/task プレゼン準備 due:金曜
/task 確定申告 due:2026-05-15
```

#### すべて組み合わせ

```
/task ブログ記事を書く @高 #学習 due:明日
/task 新機能の実装 @中 #開発 due:金曜
/task 買い物リスト作成 @低 #生活 due:今日
```

---

### レスポンス

#### 成功時

```
✅ タスクを追加しました
🔴 ブログ記事を書く 📚 学習 (期限: 2026-04-30)
```

#### エラー時

```
❌ タスク名を入力してください
使い方: /task タスク名 @優先度 #カテゴリ due:期限
```

## /tasks - タスク一覧

未完了のタスク一覧を表示します。

### 基本構文

```
/tasks [フィルター]
```

### フィルター

| フィルター | 説明 | 例 |
|-----------|------|-----|
| **(なし)** | 全タスク表示 | `/tasks` |
| **高** / **中** / **低** | 優先度フィルター | `/tasks 高` |
| **今日** | 今日が期限 | `/tasks 今日` |
| **学習** / **開発** / **生活** / **趣味** | カテゴリフィルター | `/tasks 学習` |

---

### 使用例

#### 全タスク表示

```
/tasks
```

**レスポンス:**
```
📋 タスク一覧 (5件)

🔴 高
• ブログ記事を書く 📚 学習 (期限: 明日)
• 重要な会議の準備 (期限: 今日)

🟡 中
• Flutter を勉強する 📚 学習
• API を実装する 💻 開発

🟢 低
• 本を読む 📚 学習
```

#### 優先度でフィルター

```
/tasks 高
```

**レスポンス:**
```
📋 タスク一覧 (フィルター: 高) (2件)

🔴 高
• ブログ記事を書く 📚 学習 (期限: 明日)
• 重要な会議の準備 (期限: 今日)
```

#### 今日が期限のタスク

```
/tasks 今日
```

**レスポンス:**
```
📋 タスク一覧 (フィルター: 今日) (1件)

🔴 高
• 重要な会議の準備 (期限: 今日)
```

#### カテゴリでフィルター

```
/tasks 学習
```

**レスポンス:**
```
📋 タスク一覧 (フィルター: 学習) (3件)

🔴 高
• ブログ記事を書く 📚 学習 (期限: 明日)

🟡 中
• Flutter を勉強する 📚 学習

🟢 低
• 本を読む 📚 学習
```

---

### レスポンス

#### タスクが存在する場合

優先度別にグループ化して表示

#### タスクがない場合

```
📋 タスクはありません
```

または

```
📋 タスクはありません (フィルター: 高)
```

## /done - タスク完了

タスクを完了状態にします。

### 基本構文

```
/done タスク名の一部
```

### 動作

1. 未完了タスクから**部分一致**で検索
2. 最初に見つかったタスクを完了
3. Notion で「ステータス」を「完了」に変更
4. 「完了日」に今日の日付を設定

---

### 使用例

#### 基本

```
/done ブログ
```

→ 「ブログ」を含むタスクを完了

#### より具体的に

```
/done ブログ記事を書く
```

→ 完全一致に近いほど確実

---

### レスポンス

#### 成功時

```
✅ タスクを完了しました！
🔴 ブログ記事を書く 📚 学習 (期限: 2026-04-30)
```

#### タスクが見つからない場合

```
❌ 「ブログ」に一致するタスクが見つかりませんでした
```

#### エラー時

```
❌ タスク名を入力してください
使い方: /done タスク名（部分一致）
```

---

### 注意点

#### 複数一致する場合

```
タスク:
- ブログ記事の下書き
- ブログ記事の公開

/done ブログ
→ 最初に見つかった方（下書き）が完了
```

**対策**: より具体的なキーワードを使用

```
/done 下書き  # 「下書き」を完了
/done 公開    # 「公開」を完了
```

## 実用例

### 朝のルーティン

```
# 1. 今日のタスクを確認
/tasks 今日

# 2. 新しいタスクを追加
/task 朝会の準備 @高 #開発 due:今日
/task メールチェック @中 due:今日
```

### 作業中

```
# 緊急タスクが入った
/task 緊急バグ修正 @高 #開発 due:今日

# タスク一覧を再確認
/tasks 高
```

### 作業終了時

```
# 完了したタスクをマーク
/done 朝会
/done メール
/done バグ修正

# 残タスクを確認
/tasks
```

### 週末の振り返り

```
# 未完了タスクを確認
/tasks

# 優先度を確認
/tasks 高

# 完了できるものを片付ける
/done タスク名
```

## Tips

### 効率的なタスク名

#### 良い例 ✅

```
/task ブログ記事「React hooks」を書く
/task API エンドポイント /users を実装
/task 週報を書く
```

→ 具体的で `/done` で検索しやすい

#### 悪い例 ❌

```
/task 作業
/task やること
/task タスク
```

→ 曖昧で `/done` で特定しにくい

---

### 優先度の使い分け

| 優先度 | 使いどころ |
|-------|-----------|
| **🔴 高** | 今日中に必ずやる / 緊急 |
| **🟡 中** | 今週中にやる / 通常 |
| **🟢 低** | 時間があればやる / 後回しOK |

---

### カテゴリの使い分け

| カテゴリ | 使いどころ |
|---------|-----------|
| **📚 学習** | 勉強・読書・スキルアップ |
| **💻 開発** | コーディング・技術調査 |
| **🏠 生活** | 買い物・家事・雑用 |
| **🎨 趣味** | 個人プロジェクト・趣味 |

---

### 期限の設定

```
緊急度が高い → due:今日
1週間以内 → due:金曜
具体的な締切 → due:2026-05-01
```

---

### ショートカット

Slack のショートカット機能を使うと便利:

```
# よく使うコマンドを登録
/task 日報を書く @中 #開発 → 「日報」で呼び出し
/tasks 高 → 「優先度高」で呼び出し
```

| コマンド | 用途 | 引数 | 戻り値 |
|---------|------|------|--------|
