## 使用量の見積もり

### 前提条件

- 1日あたりのタスク追加: 5件
- 1日あたりのタスク一覧確認: 7回
- 1日あたりのタスク完了: 3件

### 月間使用量（30日）

```
タスク追加: 5件 × 30日 = 150回
タスク一覧: 7回 × 30日 = 210回
タスク完了: 3件 × 30日 = 90回
──────────────────────────
合計: 450回/月
```

### サービス別影響

#### Vercel

```
450回/月 ÷ 1,000,000回/月 = 0.045%
```

**余裕**: 99.955%

#### Notion

```
450 API コール/月 ÷ (3 req/秒 × 2,592,000秒/月)
= 450 ÷ 7,776,000 = 0.0058%
```

**余裕**: 99.994%

## コスト最適化のヒント

### 1. Vercel の Cold Start を減らす

```javascript
// 定期的に ping して warm に保つ（オプション）
// vercel.json
{
  "crons": [
    {
      "path": "/api/ping",
      "schedule": "*/15 * * * *"  // 15分ごと
    }
  ]
}
```

**効果**: レスポンス時間が 1〜2秒 → 0.3秒

**コスト**: 月2,880回（無料枠の0.288%）

---

### 2. Notion API のバッチ処理

```javascript
// 複数タスクを一度に取得
const tasks = await getTasks();  // 1回の API コール

// 個別に取得するより効率的
// await getTask(id1);  // 3回の API コール
// await getTask(id2);
// await getTask(id3);
```

**効果**: API コール数削減

---

### 3. キャッシュの活用

```javascript
// タスク一覧をメモリキャッシュ（5分間）
let cachedTasks = null;
let cacheTime = 0;

async function getTasks() {
  const now = Date.now();
  if (cachedTasks && now - cacheTime < 300000) {
    return cachedTasks;
  }
  
  cachedTasks = await notion.databases.query(...);
  cacheTime = now;
  return cachedTasks;
}
```

**効果**: Notion API コール削減
